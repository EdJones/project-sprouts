What Is It?

Project Sprouts is a highly cohesive, loosely-coupled collection of features that take some of the suck out of programming.

Sprouts has been seen working on Mac, Windows and Linux and while it currently targets ActionScript, Flex and AIR development, tools can be easily created to improve any programming environment.

Sprouts is a modular development platform that takes advantage of Ruby and RubyGems to share code generators, libraries, executables and automated build tasks. It runs anywhere Ruby does, including OS X, Windows, and *nix variants.

Why would someone use Sprouts?

There are lots of reasons to use Sprouts for your projects, here are some of mine:

  • Decouple builds from development tools
  • Ensure that everyone on the team is using the exact same compilers, libraries and runtimes - by version
  • Critical tools should be platform independent
  • Avoid manually creating project files and folders over and over
  • Avoid manually adding project dependencies over and over
  • Avoid manually creating boilerplate classes, test cases and test suites
  • Want to create new exploration projects with a single command
  • Want to easily integrate projects with a Continuous Integration server
  • Want to allow anyone on the project team to easily build the project

Getting Started

To get started with Project Sprouts, you'll need Ruby (>= 1.9.2), RubyGems (>= 1.3.7) and at least some interest in the programming language Ruby and the build tool Rake.

{% highlight bash %} # Install the ActionScript 3 / Flash Bundle: $ gem install flashsdk --pre # Generate a new ActionScript 3 project: $ sprout-as3 SomeProject # Move into the new project: $ cd SomeProject # Compile and run the main project: $ rake # Congratulations - you're up and running! {% endhighlight %}

To go a little further with Test-Driven Development:

{% highlight bash %} # Generate a class, test and test suite: $ sprout-class utils.MathUtil # Compile and run the test harness: $ rake test {% endhighlight %}

And finally:

{% highlight bash %} # Show available tasks: $ rake -T {% endhighlight %} More Details

Core Concepts

Tools

A tool is any executable that has been distributed within, or referenced by, a Sprout::Specification.

Tasks

Sprouts takes advantage of the Rake build language, and brings the power of this language to non-Ruby programming environments.

Libraries

By using Sprouts, you can package shared code in source or compiled form and distribute it directly within a RubyGem or even refer to it at a different location on the internet. In either case, the library will be downloaded, cached, and made available to users of your project.
